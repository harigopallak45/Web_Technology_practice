Web Technology Notes

What is a Web?

  • Web is a system for sharing information over the internet using WWW (World Wide Web).
  • It is one of the ways to share information globally over internet connections.

Internet

  • A global network of interconnected hardware devices.
  • Connects different networks together.
  • Types of internet connections:
    • Wired
    • Wireless

Browser

  • A software used to access websites, commonly called a client in the client-server model.
  • The client is the machine or software that requests information from servers.

Web Page

  • A digital document that can be accessed via a browser.

Website

  • A collection of related web pages hosted on the internet.

Web Server

  • A powerful computer with internet access that stores websites and their related information.

Types of Web Applications

  1. Static Web Applications:
    • Displays the same information for every user.
    • No dynamic content changes.
  2. Dynamic Web Applications:
    • Displays different information based on server-side changes or user interactions.

Components of Web Applications

  1. Frontend:
    • The part of the website visible to the user.
    • Technologies: HTML, CSS, JavaScript.
  2. Backend:
    • The server-side part of a website that is not visible to users.
    • Handles data and business logic.

HTML (HyperText Markup Language)

What is HTML?

  • Stands for HyperText Markup Language.
  • It structures the content of web pages using tags.
  • HTML is called a markup language because it uses tags to define elements on a page.

HTML Syntax

  • HTML uses opening and closing tags to define elements.
    • <tagname>content</tagname>
  • Some tags are self-closing and do not require a closing tag.

Basic Structure (Boilerplate) of an HTML Page

<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>This is a heading</h1>
    <p>This is a paragraph.</p>
  </body>
</html>
  1. DOCTYPE: Declares the document type and version of HTML.
  2. html: The root element that contains the entire HTML document.
  3. head: Contains metadata and links (e.g., CSS, scripts).
  4. body: Contains the content displayed to the user.

Div Element in HTML

  • A div creates sections in a webpage and is a block element.
  • Block elements take up the full width available.
  • Use CSS to style div elements and control layout.

Float in CSS

  • Float: An attribute used to position elements.
    • float: left;: Element floats to the left side.
    • float: right;: Element floats to the right side.

Tables in HTML

  • Tables are used to display data in rows and columns.

Table Tags:

  • <table>: Defines the table.
  • <tr>: Defines a row.
  • <td>: Defines a data cell.
  • <th>: Defines a header cell.
  • colspan and rowspan:
    • Used to merge cells horizontally or vertically.

Hyperlinks in HTML

Hyperlink Tag (<a>)

  • Links one webpage to another.
  • Uses the href attribute to specify the URL. 
    <a href="https://example.com">Visit Example</a>
  • The target attribute can open the link in a new tab. 
    <a href="https://example.com" target="_blank">Open in New Tab</a>

Accessing Files with Relative Paths

  • File paths help us link or reference files from the source. To access specific folders or files, we navigate the file structure through the browser using either absolute or relative paths.
  • Relative paths are used to navigate between files in a project folder:
    • To access the parent folder, we use ../.
    • To access the parent of the parent folder, we use ../../.
    • To access a current folder, we use ./ .
    • To access a child folder, we specify the folder name followed by a slash /.

HTML Structure and Elements

  • HTML block elements take up the full width of the container (e.g., <div>, <section>, etc.).
  • Scrolling: The content will scroll automatically if it exceeds the dimensions set by CSS. Adjustments can be made using width, height, and overflow attributes.

HTML Forms

HTML forms are used to collect user input. Forms consist of various components:

  • Label Component: The <label> element describes an input field.
  • Input Component: The <input> element allows the user to enter different types of data, such as:
    • type="text": Text input.
    • type="password": Password input.
    • type="number": Numeric input.
    • type="email": Email input.
    • type="date": Date input.
    • type="file": File upload.
  • Submit Button: The button triggers form submission.
  • Placeholders: Provide hints inside input fields using the placeholder attribute.

Special Input Attributes

  • Required: Use required to make a field mandatory.
  • Readonly: Use readonly to prevent modification of the field.
  • Disabled: Use disabled to make an input field uneditable.

Dropdown Lists

  • Use the <select> element to create a dropdown list.
  • The <option> tag specifies options inside the dropdown.
Example: 
<select>
    <option value="1">Option 1</option>
    <option value="2">Option 2</option>
</select>

Semantic HTML

  • Semantic tags define the content clearly for both developers and browsers, improving readability.
  • Examples:
    • <header>: Defines a section header.
    • <nav>: Defines navigation links.
    • <article>: Defines self-contained content.
  • Non-semantic tags (e.g., <div>, <span>) do not give clear meaning about their content.

Lists in HTML

  • Ordered Lists (`<ol>`): Display items in a numbered list.
  • Unordered Lists (`<ul>`): Display items with bullet points.
  • Description Lists (`<dl>`): Used for terms and their descriptions.
Example of an ordered list: 
<ol>
<li>First item</li>
<li>Second item</li>
</ol>

CSS (Cascading Style Sheets)

  • Cascading refers to the flow and application of styles from different sources (e.g., internal, external, inline) to each HTML element.
  • Internal CSS: Defined within the <style> tag inside the html document.
  • External CSS: Defined in separate `.css` files and linked to the HTML using the `` tag.
  • Inline CSS: Styles are applied directly to HTML elements using the `style` attribute.
Example of linking an external CSS file: 
<link rel="stylesheet" href="styles.css">

External CSS Best Practices

Using external CSS is the preferred method because it:

  • Ensures better performance by separating content from presentation.
  • Makes the HTML cleaner and easier to maintain.
  • Allows reuse of the same stylesheet across multiple HTML pages.
